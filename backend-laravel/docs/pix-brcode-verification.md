# Pix BR Code verification

Date: 2026-05-16

## What changed

- The backend no longer returns an internal Nexora reference as if it were a bank Pix code.
- Support contributions return the requester's registered random Pix key, the EVP key generated by the requester's bank and stored encrypted in Nexora.
- `NEXORA_ADMIN_PIX_KEY` is used for administrative fees only. Normal support transfers use the requester's encrypted `pix_cipher`.
- Normal support instructions intentionally do not generate a static BR Code payload, because BR Code includes merchant/receiver fields that can expose the receiver name inside the app.
- User Pix keys must be random EVP UUID keys. CPF, CNPJ, phone, and email Pix keys are not accepted for user registration.
- The super admin CPF can be synchronized from `NEXORA_SUPER_ADMIN_CPF` for CPF login.

## Important limitation

The app can hide the receiver name inside Nexora by copying only the random Pix key. A bank app can still resolve and show the bank account holder after the donor pastes the key, because the bank must confirm the real receiver before payment.

## Status flow

- `Aguardando revisao`: user or request is waiting for admin approval.
- `Aberto`: support request can receive Pix contributions.
- `Aguardando comprovantes`: Pix instruction was generated; sender and receiver still need to upload transaction ID plus receipt image.
- `Completo`: request amount was fully allocated.
- `Validado`: admin reviewed both sides and confirmed the Pix.
- `Retornado`: admin confirmed the later return/repayment step.
- `Recusado`: request was rejected.
- `Bloqueado`: user cannot act until admin changes status.
